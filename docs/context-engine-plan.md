# 인물·서사 컨텍스트 엔진 계획 (CX)

> 2026-07-15. pipeline-plan.md의 상위 확장. B2B 전제(이미 활성화된 채널·프로그램을 가진 방송사)에서 나오는
> 결정적 이점: **출연자 정보와 회차·배포 데이터가 처음부터 존재**한다. 이를 축으로 영상 이해를
> "익명 장면 분석"에서 "인물·서사 기반 분석"으로 끌어올린다.

## 한 문장 요약

프로그램 출연자를 등록해 두면(나는솔로 23기: 출연자 12명 + MC 3명), 영상 분석 시 "20대 남성"이
아니라 **"23기 영숙"으로 확정 식별**되고, 회차가 진행될수록 인물별 서사·시청자 반응·성과가 쌓여,
"다음 화에서 어떤 인물의 어떤 장면을 쇼츠로 만들어야 터지는가"를 데이터로 아는 시스템.

## 루프 전체 그림

```
[캐스트 등록] → [프레임: 인물 확정 + 배경/상황] → [장면 요약] → [시간대 롤업(서사 타임라인)]
                                                                        ↓
[인물 KB 갱신] ←── [배포 성과·댓글 반응(인물별)] ←── [배포] ←── [쇼츠/하이라이트 선정]
      └──────────────── 다음 회차 분석·선정의 컨텍스트로 주입 ────────────────┘
```

## 계층별 설계

### CX-1. 캐스트 레지스트리 (인물 등록)

- 프로그램·기수(시즌) 단위로 출연자 등록: 이름/역할(출연자·MC)/대표 얼굴 이미지 수 장.
- 방송사는 이 정보를 이미 갖고 있음 → 운영자 입력 UI + 1화 분석 후 "미등록 얼굴 클러스터"를 보여주고 이름 붙이는 반자동 온보딩.
- 얼굴 식별 기술:
  - **1차 (관리형): Gemini 멀티모달** — 등록된 대표 얼굴들을 레퍼런스로 프롬프트에 넣고 프레임 속 인물 매칭. 출연자 15명 수준의 닫힌 후보군이라 정확도 유리 (오픈셋이 아니라 "이 15명 중 누구" 문제).
  - **2차 (물량 증가 시): 얼굴 임베딩 + pgvector**(Cloud SQL 확장) 검색으로 전환. ⚠️ InsightFace 등 유명 모델은 코드 MIT여도 **모델 가중치가 비상업**인 경우가 많음 — 도입 시점에 라이선스 재조사 필수.
- ⚠️ 법무: 얼굴 인식은 개인정보(민감정보) 처리 — 방송사와의 계약에 출연자 초상 데이터 처리 근거 명시 필요. B2B라 방송사가 출연 계약상 권리를 보유하는 구조를 활용.

### CX-2. 장면 이해 (프레임 → 장면 → 시간대 롤업)

- 샷 분할(ffmpeg 씬 검출) → 샷당 대표 프레임 1~2장만 분석 (프레임 전수 분석 금지 — 비용 상한).
- 샷 단위 Gemini 호출에 **3중 컨텍스트 주입**이 핵심:
  1. 확정 인물 ID (CX-1 결과: "영숙, 영호")
  2. 해당 구간 전사 텍스트 (B의 transcript — 화자 분리를 얼굴과 교차해 "누가 말했는지"까지 귀속)
  3. 인물 KB 요약 (지금까지의 서사: "영숙은 3화에서 영호에게 호감 표시")
  → 출력: 구조화된 장면 레코드 `{시각, 공간, 등장인물, 행동, 요약문("영숙이 영호에게 노래를 불러주고 산책 중")}`
- 롤업: 장면 레코드들을 시퀀스(수 분)→회차 단위로 LLM 요약 → **서사 타임라인** 테이블. 회차 전체 시놉시스가 부산물로 나옴 (메타데이터 I에도 재사용).

### CX-3. 인물 지식 베이스 (KB)

인물별로 회차·배포가 진행될 때마다 갱신되는 누적 데이터:

| 필드 | 원천 | 갱신 시점 |
|------|------|----------|
| 기본 프로필 (이름·기수·역할) | 운영자 등록 | 온보딩 |
| 회차별 행동 로그 | CX-2 장면 레코드 | 회차 분석 완료 시 |
| 페르소나/이미지 태그 ("직진남", "밈 캐릭터") | 장면 로그 + 댓글에서 LLM 추출 | 회차·배포 후 |
| 시청자 반응 (인물별 댓글 언급량·감성·기대 포인트) | 배포 클립 댓글 분석 (J 확장 — 댓글 요약을 인물 단위로 구조화) | 배포 후 지속 |
| 인물별 성과 (그 인물 중심 쇼츠의 조회·완주율) | 채널 분석 파이프라인 metrics × 클립 등장인물 태그 | 배포 후 지속 |

- 저장: `person`, `person_episode_log`, `person_reaction` 정규 테이블. 클립에는 등장인물 태그가 붙음 (성과 귀속의 키).

### CX-4. 선정 통합 (쇼츠/하이라이트 결정)

기존 점수 구조(pipeline-plan.md E 확장)에 서사·인물 축 추가:

```
최종점수(후보, 채널) = [ 비전 0.70 + 로컬(훅) 0.30 ]           ← 기존
                      × 채널 적합 계수                          ← 기존
                      × 인물·서사 계수 (신규)
```

인물·서사 계수의 입력:
- **인물 화제성**: 지난 배포에서 이 인물 쇼츠의 성과 배수 + 댓글 언급량 추이
- **기대 매칭**: 댓글에서 추출한 기대 포인트("영숙 답장 언제 나옴?")와 이번 후보 장면의 매칭 — 시청자가 기다리던 장면은 강한 부스트
- **서사 완결성**: 장면 레코드 기준으로 후보 구간이 행동의 시작~끝을 포함하는지 (문장 단위 스냅 D를 장면 단위로 확장)
- 하이라이트(롱폼)와 쇼츠는 같은 서사 타임라인에서 길이·완결성 기준만 달리해 선정.

### CX-5. 환류

- 배포 → 인물별 성과·댓글이 KB 갱신 → 다음 회차의 CX-2 프롬프트 컨텍스트와 CX-4 계수에 자동 반영.
- 채널 분석 파이프라인(pipeline-plan.md)의 (프로그램, 채널, 훅, 길이) 차원에 **인물** 차원 추가 — "이 채널에서는 영숙 클립이 평균 2.3배" 같은 단위로 축적.

## 발명신고서와의 관계 (특허 확장 후보)

구성 B(단일 원천 공유)·E(융합 평가)·J(성과 환류)의 확장이며, 다음이 신규 청구 후보로 보임:
1. 사전 등록 캐스트 기반 폐쇄 후보군 인물 확정 식별 + 화자 분리 교차에 의한 발화 귀속
2. 인물 KB(서사·반응·성과)의 회차 간 누적과 이를 선정 계수로 환류하는 폐루프
3. 댓글 기대 포인트와 미방영 장면 매칭에 의한 선정 부스트
→ 변리사 검토 목록에 추가 권장.

## 마일스톤 (CX 트랙 — AP 트랙과 병행)

- **CX1** (AP2 이후): 캐스트 등록 UI + 샷 분할 + Gemini 인물 확정 식별. 클립 등장인물 태그부터 시작 (성과 귀속의 최소 단위).
- **CX2** (AP3와 병행): 장면 레코드 + 서사 타임라인. 회차 시놉시스 자동 생성.
- **CX3** (AP4 이후): 인물 KB + 댓글 인물별 구조화. 
- **CX4** (AP5): 인물·서사 계수 선정 통합 + 기대 매칭 부스트 + 인물 차원 성과 환류.

## 비용 통제

- 장면 이해는 샷당 1~2프레임 + 프록시 해상도 + 배치 호출. 90분 회차 ≈ 샷 수백~1천 개 수준 → 회차당 호출 상한을 config로 고정 (발명신고서의 비용-계단 원칙 유지).
- 인물 식별은 "등장 샷"에만 (씬 검출 + 얼굴 존재 프레임 필터 후 호출).
- KB 요약은 프롬프트에 전문이 아닌 압축 요약본만 주입 (토큰 상한 고정).
