## 🔴 Critical (P0) — 당장 구현 안 하면 CapCut 따라잡기 불가능

### P0-1. 단일 트랙 타임라인 — 멀티트랙 부재

**현재:** 타임라인에 오디오 파형 1개 트랙만 표시됨. 타이틀·요소·자막은 별도 캔버스에서만 조작 가능.

**CapCut:** 비디오 트랙 + 오디오 트랙 + 텍스트 트랙 + 오버레이 트랙이 **세로로 쌓여** 한눈에 모든 레이어의 시간 배치가 보임.

```
[현재 STEPD]                          [CapCut]
┌─────────────────────┐              ┌─────────────────────┐
│ ~~~~~파형~~~~~      │              │ 🎬 비디오 트랙       │
│ [trim ████ window]  │              │ 🔤 텍스트 트랙       │
└─────────────────────┘              │ 🔊 오디오 트랙       │
                                     │ ⭐ 오버레이 트랙      │
                                     └─────────────────────┘
```

**영향:** 사용자는 "이 타이틀이 언제 나타나고 사라지는지"를 타임라인에서 전혀 알 수 없음. 요소 추가 시 인/아웃 타이밍 제어 불가.

**필요 작업:**
- 타임라인을 세로 다중 트랙 구조로 변경
- 각 트랙: 비디오(원본), 텍스트(타이틀), 오버레이(CTA·스티커·화살표), 오디오(볼륨)
- 각 오버레이 요소에 `startTime`/`endTime` 속성 추가 → EditorElement 확장

---

### P0-2. 트림 핸들이 타임라인 바깥에 있음

**현재:** 트림 IN/OUT 조정이 **타임라인 아래쪽 range slider + 숫자 입력**으로만 가능. 타임라인 트랙 위의 트림 영역(초록색)은 표시만 되고 드래그 불가.

```
[현재]
│ ~~~~파형 [████트림███] ~~~~ │  ← 클릭해도 아무 일도 안 일어남
│ IN [====슬라이더====] 0:05  │  ← 여기서 조정해야 함
│ OUT[====슬라이더====] 0:35  │
```

**CapCut:** 트림 영역 양 끝에 **드래그 핸들**이 있고, 타임라인 위에서 직접 잡아당겨 조정. 별도 슬라이더 없음.

**영향:** 직관성 ↓↓. "타임라인 보면서 트림 조정"이라는 기본 동작이 분리되어 있음. UI 기획에서 정의한 "인라인 트리밍(#6)"이 여기 해당.

**필요 작업:**
- 트림 영역에 드래그 핸들 추가 (흰색 바 + 좌우 화살표)
- range slider 제거 또는 보조로 변경
- 드래그 중 프레임 썸네일 팝오버

---

### P0-3. 클립 분할(Split) 기능 없음

**현재:** 편집기는 하나의 연속 구간(trimIn~trimOut)만 다룸. 영상 중간을 잘라서 앞/뒤 두 클립으로 나누는 기능 없음.

**CapCut:** 타임라인에서 재생 헤드 위치에서 `Ctrl+B` (또는 분할 버튼) → 클립이 두 개로 쪼개짐. 각각 독립적으로 트림/삭제/이동 가능.

**영향:** AI가 추천한 구간(예: 3:00~4:30)의 중간 30초만 빼고 앞뒤를 버리고 싶을 때 불가능. 지금은 accept or reject 2택.

**필요 작업:**
- "분할" 버튼/단축키 추가
- 현재 재생 위치에서 클립을 둘로 나누고 각각 트림 가능하게

---

### P0-4. Undo/Redo 없음

**현재:** `useState`로 단일 `EditorState` 관리. 이전 상태 스택 없음. 실수로 타이틀 삭제하거나 요소 위치 망가뜨리면 되돌릴 방법 없음.

**CapCut:** `Ctrl+Z` / `Ctrl+Y` 무제한 undo/redo.

**영향:** 사용자 신뢰도 ↓↓. 편집기는 실수하기 쉬운 도구인데 되돌리기가 안 되면 불안해서 못 씀. **P0 중에서도 가장 시급.**

**필요 작업:**
- `useRef<EditorState[]>` 히스토리 스택 (과거 + 미래)
- `Ctrl+Z` / `Ctrl+Y` 단축키
- 상단 툴바에 ↶ ↷ 버튼

---

### P0-5. 타임라인 줌 인/아웃 없음

**현재:** 타임라인 파형이 고정 너비. 42분짜리 영상에서 0.1초 단위로 트림 조정하려면 슬라이더로 불가능.

**CapCut:** 마우스 휠/핀치로 타임라인 확대/축소. 줌 레벨에 따라 초 단위 눈금 자동 변경.

**영향:** 정밀 편집 불가. "여기서 0.3초만 더" 같은 조정이 range slider로는 물리적으로 안 됨.

**필요 작업:**
- 타임라인 `scale` 상태 추가 (예: 1x, 2x, 4x, 8x)
- 마우스 휠 이벤트 → 줌
- 눈금자(초 단위) 표시 + 줌에 따라 단위 변경

## 🟡 Important (P1) — 있으면 극적 개선

### P1-1. 속도 램핑 (Speed Ramping) 없음

**현재:** 0.5× · 1× · 1.5× · 2× 중 하나 선택 (전체 구간 균일).

**CapCut:** 속도 커브 — "처음 1초는 100%, 중간 2초는 200%, 마지막은 50%" 같은 가변 속도.

**영향:** 바이럴 쇼츠의 핵심 기법인 "슬로모션 → 갑자기 빨라짐" 연출 불가.

**필요 작업:**
- `EditorState`에 `speedCurve: {time, speed}[]` 추가
- 타임라인에 속도 커브 시각화

---

### P1-2. 전환 효과 (Transition) 없음

**현재:** 분할 기능도 없으므로 트랜지션은 당연히 없음.

**CapCut:** 페이드 인/아웃, 크로스 디졸브, 와이프 등 다양한 전환.

**영향:** 쇼츠 시작/끝이 툭 끊김. 기본적인 페이드 인/아웃만 있어도 완성도 ↑↑.

**필요 작업:**
- `trimIn`/`trimOut`에 fadeIn/fadeOut duration 속성 추가
- ffmpeg 렌더 시 fade 필터 적용

---

### P1-3. 오디오 볼륨 제어 없음

**현재:** 오디오 파형은 **표시만** 됨. 볼륨 조절, 페이드, 덕킹 불가.

**CapCut:** 오디오 트랙에서 볼륨 엔벨로프 조정, 페이드 인/아웃, 음악/BGM 볼륨 믹싱.

**영향:** 원본 소리가 너무 크거나 작아도 조절 못 함. BGM 추가 시 음성 묻힘.

**필요 작업:**
- 볼륨 슬라이더 (0%~200%)
- 오디오 페이드 인/아웃
- (장기) BGM 트랙 추가

---

### P1-4. 요소 시간 제어 없음

**현재:** CTA 버튼·스티커·화살표·말풍선은 **캔버스에 고정**. 등장/퇴장 타이밍 없음.

**CapCut:** 각 오버레이 요소에 지속 시간(duration)과 인/아웃 애니메이션 설정 가능.

**영향:** "3초 후에 CTA 버튼 나타나기" 같은 기본 패턴 불가.

**필요 작업:**
- `EditorElement`에 `startTime`, `endTime`, `animation` 속성 추가
- 멀티트랙 타임라인에 오버레이 트랙 표시

---

### P1-5. 키프레임 애니메이션 없음

**현재:** 모든 오버레이는 정적 위치.

**CapCut:** 요소의 위치·크기·투명도를 시간에 따라 키프레임으로 애니메이션.

**영향:** "타이틀이 왼쪽에서 오른쪽으로 슬라이드" 같은 모션 연출 불가.

**필요 작업:**
- 요소별 키프레임 배열 `{time, x, y, scale, opacity}[]`
- 타임라인에 키프레임 다이아몬드 마커

---

### P1-6. 필터·이펙트 없음

**현재:** 컬러 그레이딩, 필터, 뷰티 이펙트 전무. `bg` 색상만 변경 가능.

**CapCut:** 100+ 비주얼 필터, 뷰티(피부 보정), LUT, 색보정 휠.

**영향:** 쇼츠의 비주얼 퀄리티가 원본에 종속. 색감 보정 불가.

**필요 작업:**
- (단기) 기본 필터 5종 (warm, cool, vivid, cinematic, bw)
- ffmpeg colorbalance/eq 필터 연동

## 🎯 STEPD만의 차별화 포인트

CapCut은 범용 편집기지만, STEPD는 **AI 분석 데이터와 편집기가 연동**된다는 점이 무기:

1. **AI 구간 마커를 타임라인에** — appeal 점수, 장면 전환, 자막 밀도를 타임라인에 컬러 히트맵으로 오버레이
2. **자막 기반 내비게이션** — 타임라인이 아닌 대사 검색으로 이동 ("유재석이 웃는 장면" 검색 → 점프)
3. **Vision 분석 → 편집 제안** — "이 구간에 CTA 넣으면 좋습니다" AI 추천
4. **배포처 최적화 미리보기** — Shorts/Reels/TikTok별 1:1 미리보기가 이미 구현됨 (CapCut은 이거 없음)

## 📏 현재 EditorState 확장 필요 항목

```typescript
// 기존 EditorState를 확장 (presets.ts)
interface EditorStateV2 extends EditorState {
  // P0: undo 히스토리
  _history?: { past: EditorState[]; future: EditorState[] };
  
  // P0: 클립 분할
  segments?: { trimIn: number; trimOut: number }[];
  
  // P0: 줌
  timelineScale?: number; // 1x~16x
  
  // P1: 오디오
  volume?: number;       // 0~200 (%)
  fadeIn?: number;       // seconds
  fadeOut?: number;
  
  // P1: 속도 램핑
  speedCurve?: { time: number; speed: number }[];
  
  // P1: 필터
  filter?: 'none' | 'warm' | 'cool' | 'vivid' | 'cinematic' | 'bw';
}

// P1: 요소 시간 제어
interface EditorElementV2 extends EditorElement {
  startTime?: number;
  endTime?: number;
  animation?: 'fade' | 'slide' | 'pop' | 'none';
  keyframes?: { time: number; x: number; y: number; scale: number; opacity: number }[];
}
```
