# STEP-D 레포 분석 보고서 — 투자 관점 한국형 B2B SaaS 진단

> **분석일:** 2026-07-17  
> **레포 분석 기준:** 소스코드 9개 핵심 파일 + 마스터 빌드플랜(442줄) + 파이프라인 계획서 등 문서  
> **투자 관점 판단 기준:** 기술차별성 / 시장적합도 / 확장성 / 실행력 / IP보호 가능성

---

## 1. 제품 정체성 — 한 줄 요약

**"방송사·MCN이 긴 영상을 올리면 AI가 알아서 Shorts·Reels·SMR 맞춤형 숏폼을 추천해주고, 운영자는 승인만 누르면 자동 편집·자막·배포·성과분석까지 이어지는 B2B SaaS"**

Opus Clip이 글로벌 범용이라면, STEP-D는 **한국 방송·예능에 특화된 엔드투엔드 B2B 플랫폼**이다.

---

## 2. 기술 아키텍처 — 요약

| 계층 | 기술 스택 | 배포 |
|------|----------|------|
| **프론트엔드** | Next.js 16 (App Router) + React 19 + Tailwind v4 + TypeScript | Vercel (`stepd.stepai.kr`) |
| **백엔드 API** | Hono (Node.js) + PostgreSQL 16 + GCS | Cloud Run (us-central1) |
| **워커 (AI 실행)** | TypeScript (worker.ts) → Python spawn (`core/analyze.py`) | GCE VM e2-small (us-central1-a) |
| **큐잉** | 자체 `job_queue` 테이블 (PostgreSQL `FOR UPDATE SKIP LOCKED`) | Cloud SQL |
| **AI 모델** | Gemini 2.5 Flash (Vertex AI, asia-northeast3 서울 리전 고정) | Vertex AI (관리형, GPU 무운영) |
| **영상 처리** | ffmpeg (트림·인코딩·자막 번인) + PySceneDetect (장면분할) | 워커 VM |

**핵심 설계 원칙:**
- **GPU 자체 운영 제로** — 모든 AI 추론은 Vertex AI 관리형 API로 처리. 인프라 비용이 예측 가능하고 확장 용이.
- **데이터 레지던시** — 개인정보(얼굴·음성·방송 영상)를 서울 리전(asia-northeast3)에서만 처리 → PIPA 규제 선제 대응.
- **렌더 최종 1회 원칙** — 분석·선정·편집·프리뷰는 전부 메타데이터로만(무렌더), 최종 확정 시에만 ffmpeg 인코딩. 회차당 비용이 상수로 수렴.

## 4. Opus Clip 대비 차별화 분석

| 축 | Opus Clip | STEP-D (계획+실제) |
|----|-----------|---------------------|
| **언어** | 영어 최적화 | **한국어 특화** — 종결어미 스냅, 훅 사전, 예능 말맛 보존 |
| **STT 품질** | 범용 Whisper | Gemini 2.5 Flash + faster-whisper 폴백 (한국어 고유명사 보존 검증됨) |
| **장르 지능** | 범용 (바이럴 점수) | **7종 장르팩** (예능/토크/스포츠 등) + 프로그램별 프로파일 |
| **추천 방식** | 단일 랭킹 | **(후보 × 채널) 매트릭스** — 같은 영상이라도 Shorts/Reels/SMR별 다른 구간 상위 |
| **컨텍스트** | 영상 단위 | **프로그램·채널·인물 누적 컨텍스트** (회차 쌓일수록 더 정확) |
| **배포** | 수동 다운로드 | YouTube/Meta/SMR 원클릭 배포 + 성과 자동 회수 |
| **데이터 주권** | 글로벌 클라우드 | **서울 리전 고정** — PIPA, 방송사 개인정보 규제 대응 |
| **편집기** | 내장 | OpenCut MIT 발췌 + 자체 골격 (서버 렌더, B2B 모델에 부합) |
| **과금 모델** | 사용량 기반 | **회차당 AI 비용 상수** — B2B 원가 설계 근거 확보 |

**핵심 차별점 = B2B 컨텍스트 자산.** Opus Clip은 "이 영상에서 재밌는 구간"을 찾지만, STEP-D는 "이 프로그램의 23기 영숙이가 나오는 이 구간이 이 채널에 맞는가"를 안다. 방송사는 이미 활성 채널·프로그램·출연자 데이터를 보유하고 있어, 진입 즉시 차별화 발휘 가능.

## 5. 투자 관점 평가

### 강점 (5대 투자 포인트)

| # | 강점 | 근거 |
|---|------|------|
| 1 | **검증된 기술 코어** | AI 파이프라인(STT→장면→비전→추천→채택)이 실제 Vertex AI로 구동되고, 추천 결과가 보드에 직결되어 운영 가능. "PPT만 있는 회사"가 아님. |
| 2 | **특허 가능 차별화** | 5대 발명신고 축 (종결어미 스냅 / 비용-계단 / STT 단일원천 / 채널매트릭스 / 컨텍스트엔진) — 한국어·한국 방송에 특화된 자체 IP. |
| 3 | **B2B 원가 구조 설계 완료** | AI 비용이 회차당 상수(비전 평가 수 상한 고정) → 과금 모델·마진 예측 가능. |
| 4 | **빠른 시장 진입 가능** | Opus Clip 한국어 품질 이슈 → 실수요 발생 중. 방송사·MCN은 이미 유사 도구에 월 수백만원 지출. |
| 5 | **규제 선제 대응** | 데이터 레지던시(서울 리전), PIPA 민감정보 처리 근거 설계 → 글로벌 경쟁사 대비 차별화. |

### 약점·리스크

| # | 리스크 | 완화 방안 |
|---|--------|----------|
| 1 | **렌더링 파이프라인 지연** | 채택→확정→렌더 이연 아키텍처는 설계 완료. 구현 우선순위 조정 필요. |
| 2 | **훅 사전·종결어미 미구현** | 한국어 특화 알고리즘은 자체 IP 핵심 — 빠른 PoC 검증 긴요. |
| 3 | **컨텍스트 엔진 미착수** | 계획 완료 (5계층 설계), 게이트(PoC 정확도 90%+) 필요. |
| 4 | **멀티플랫폼 배포 스텁** | YouTube 실업로드는 인프라 갖춤. Meta Graph API는 앱심사 리드타임. |
| 5 | **1인/소규모 팀** | 핵심 IP(AI 파이프라인)는 이미 동작. UI/편집기는 오픈소스 부품(MIT) 활용 계획. |

## 7. 코드 품질 & 엔지니어링 평가

- **아키텍처 명확성:** monorepo (pnpm workspace) 구조, `apps/web` + `apps/server` + `core/` + `admin/`로 책임 분리 완료.
- **문서화:** 442줄 마스터 빌드 플랜에 "정본/폐기 문서" 구분, 불변식(렌더 1회), 갭 분석까지 포함. `CLAUDE.md`에 컨텍스트 파일 존재. 투자 실사 수준 문서화.
- **복원력 설계:**
  - STT 이중화 (Gemini → faster-whisper 폴백)
  - 장면 분석 Resume 지원 (에러 시 재시도, 완료된 작업 스킵)
  - 자체 큐: dedupeKey + 지수 백오프 + maxAttempts 5
  - GCS 직접 업로드 (Cloud Run 메모리 우회)
- **라이선스 위생:** AGPL(YOLO)·GPL(Etro)·유료(Remotion) 사전 배제, MIT(OpenCut)·Apache(MediaPipe) 선정 완료.
- **한국어 특화 인사이트:** Gemini가 Google STT보다 한국어 고유명사 인식률 우수함을 실측하여 기본값 채택. 말맛 보존(반말·유행어 살림) 프롬프트에 반영.

## 9. 종합 판정

> **투자 유치 가능성: 높음 (조건부)**

**근거:**
1. **기술이 실제로 동작한다** — STT→장면→비전→추천→보드까지 프로덕션 배선 완료. PoC 아닌 MVP 레벨.
2. **시장 타이밍이 좋다** — Opus Clip 한국어 이슈로 틈새 발생, 숏폼 시장은 성장 중.
3. **차별화가 방어 가능하다** — 한국어 특화 알고리즘 + B2B 컨텍스트 자산은 글로벌 도구가 쉽게 따라올 수 없는 영역.
4. **아키텍처가 B2B에 맞게 설계됐다** — 원가 예측 가능, 규제 대응, 라이선스 위생.

**보완 필요:**
- 렌더링/배포 완주 시연이 투자 유치의 결정적 임계점
- 1개 이상의 실제 방송사/MCN 레퍼런스 필요
- 팀 규모(1인)에 대한 설득력 있는 확장 계획 필요
