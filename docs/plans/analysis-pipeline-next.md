## 2. 작업 (우선순위 순)

### P1. 종결어미 스냅 — `core/recommend.py`
**왜 1순위인가**: 결과 품질이 가장 눈에 띄게 좋아지는데 **의존성이 없다.** G2(단어 타임스탬프)를
기다릴 필요가 없다 — 자막 세그먼트가 이미 **발화 단위**라, 그 경계가 곧 문장 경계에 가깝다.

- 후보의 `start`/`end`를 ±2.5초 안에서 가장 가까운 **발화 경계**로 당긴다.
- 종결어미 판정: 세그먼트 텍스트 끝이 `다|요|죠|까|네|군요|잖아|는데` (+ 선택적 `.!?`) 로 끝나면 우선순위 가점.
  종결어미가 없으면 발화 경계만으로 스냅(현행보다는 낫다).
- 시작 경계는 **발화 시작**에 스냅하되, 앞 0.3초 여백(호흡)을 남긴다.
- 삽입 위치: `validate_shorts`(:446) 직전 — 클램프·검증 전에 경계를 확정해야 길이 검증이 의미를 갖는다.
- **검증**: 기존 `analysis.json` 몇 건에 대해 스냅 전/후 경계가 자막 세그먼트 경계와 얼마나 일치하는지 비율로 측정. 사람 눈 검사는 Lab에서 구간 재생.

### P2. 채널 매트릭스 UI 노출 — `apps/web`
**왜 2순위인가**: 엔진이 이미 값을 만든다. **프론트만 붙이면 되는 가장 싼 가치**다.

- `channel_scores`는 `content_analysis.data.shorts[]`에 있고 `recFromShort`가 추천 엔티티로 옮길 때
  이미 `channelScores`로 넘긴다(`content-pipeline.ts:225-256`). 서버 작업 없음.
- 추천 보드에 채널 축(탭 또는 열)을 추가해 "이 구간은 Shorts 1위, SMR 5위" 식으로 보이게 한다.
- 채택 시 `pickTargetChannel`이 이미 최고점 채널을 `targetChannel`로 넣으므로 편집기까지 연결돼 있다.

### P3. 단어 단위 타임스탬프 — **필요한 곳에만**
**핵심 판단**: 회차 전체에 단어 타임스탬프를 만들 필요가 **없다.** 필요한 건 최종 후보 ~10개의
구간(각 30~60초)뿐이다. 전체를 whisper로 돌리면 비싸고 느리지만, **선정된 구간만** 돌리면
회차당 5~10분 분량이라 CPU로도 감당된다. §0의 비용-정밀도 계단을 정렬에도 그대로 적용하는 것.

- `recommend` 확정 후, 선정 구간만 잘라 faster-whisper(int8 CPU)로 **강제 정렬** → 단어 타임스탬프.
- 산출물은 자막 번인(`buildEditorAss`)과 편집기 싱크 조정에 쓴다.
- P1이 이미 발화 경계를 잡아주므로, 이건 **경계 문제 해결이 아니라 자막 품질** 작업이다.

### P4. 학습 재랭킹 — `apply_learned_rerank`
[channel-domain-adaptation.md](channel-domain-adaptation.md)에 상세. 요약하면:
데이터(Lab 매칭 + 채택/거절 + 리텐션) → 오프라인 데이터셋 → LightGBM → **오프라인 A/B 게이트 통과 시에만** 편입.
현재 Lab 매칭이 쌓이는 중이라 데이터 단계가 진행형이다.

## 3. 순서와 의존성

```
P1 종결어미 스냅 ──────────────► 품질 즉시 개선 (의존성 없음)
P2 채널 매트릭스 UI ───────────► 엔진 완료, 프론트만
P3 구간 한정 강제정렬 ─────────► P1 이후 (경계 확정 후 그 구간만 정렬)
P4 학습 재랭킹 ────────────────► 데이터 축적 후 (진행 중)
```

P1·P2는 **서로 독립**이라 동시에 가능하다. P3은 P1 뒤가 자연스럽고, P4는 데이터가 모일 때까지 대기.

---

## 4. 하지 않을 것 (범위 밖 명시)

- **훅 사전 재작성** — 이미 8종이 있고 프로파일·채널 프리셋에 배선돼 있다. 카테고리 조정은 데이터를 보고.
- **비전 파이프라인 교체** — 사전필터 계단이 비용을 상수로 묶고 있다. 정확도 불만이 데이터로 확인되기 전엔 손대지 않는다.
- **STT 공급자 전면 교체** — Gemini 경로가 프로덕션에서 안정적이다. P3은 whisper를 *부분적으로* 쓰는 것이지 교체가 아니다.

## 5. 리스크

- **P1 과교정**: 종결어미가 드문 콘텐츠(예능 리액션·음악)에서 스냅이 오히려 경계를 망칠 수 있다.
  → 스냅 거리 상한(±2.5초)과 "종결어미 없으면 발화 경계만" 폴백으로 제한. 장르별 on/off는 프로파일로.
- **P3 비용 오판**: 구간 한정이라도 whisper CPU는 느리다. 회차당 실측 후 판단 — 느리면 자막 싱크는
  편집기에서 수동 조정으로 남긴다(현행 동작).
- **문서 드리프트**: 이번에도 "미구현" 표기가 셋이나 틀렸다. 계획 문서는 착수 직전에 코드로 재확인할 것.
