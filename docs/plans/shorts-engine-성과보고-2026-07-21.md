## 한 줄 결론

**"롱폼에서 쇼츠를 잘 뽑는가"를 숫자로 채점하는 체계를 세웠고, 엔진의 구조적 결함(너무 짧게 자름·대사 중간 끊김)을 찾아 고쳤다.** "채널 학습이 성과를 더 올리는가"는 방법론을 실증했고 지금 다중 홀드아웃으로 통계 측정 중이다.

---

## 1. 목표 — 무엇을 증명하려 했나

> 채널이 올린 긴 영상에서 우리 엔진이 **실제 편집자가 고른 것과 같은 구간**을 뽑아내는가.
> 그리고 그 채널의 과거 데이터로 학습하면 **더 잘 뽑게 되는가**.

주관("좋아 보인다")이 아니라 **숫자**로 답하는 것이 핵심 원칙이었다.

---

## 2. 성과 — 이룬 것

### 2-1. 측정 체계 — "잘 뽑는가"를 숫자로 ✅

채널이 **실제 발행한 숏폼**을 정답지로 삼았다. 사람이 고른 것이므로 가장 신뢰할 수 있는 정답이다.

| 만든 것 | 역할 |
|---|---|
| 오디오 매칭 (`align.py`) | 발행 숏폼이 롱폼 **어느 구간**에서 잘렸는지 소리로 역추적 (Gemini 없이 CPU) |
| 채점 (`evaluate.py`) | 추천 구간 vs 정답 구간을 **Hit@N·경계오차**로 채점 (IoU≥0.5) |
| 성과 정규화 | **조회수 절대값 금지** — ±90일 동시기 채널 중앙값 대비 배수. 3년 전 50만과 지난달 20만을 같은 잣대로 비교하는 오류를 구조적으로 차단 |

매칭 정확도 실측: 진짜 매칭 0.886~0.998 vs 무관 0.403~0.601 → 간격이 넓어 오탐·미탐 여유. 합성 정답 재현 오차 **4ms**.

### 2-2. 엔진 개선 — 확실히 고친 것 ✅

**① 너무 짧게 자르던 문제 (배포 완료)**
- 첫 측정에서 엔진 **Hit@5 = 0.00**. 원인은 "못 찾아서"가 아니라 정답 구간을 찾고도 **4~8초로 잘라** IoU가 무너진 것.
- 프롬프트 모순("펀치라인만/짧게")을 "셋업 포함 30~60초"로 재작성 + 30초 하한 확장.
- → 단일 측정에서 **Hit@5 0.00 → 0.67** (재현율 0/3 → 2/3). 추천 길이 8초 → 45초.

**② 대사 중간에서 시작/끊기던 문제 (이번 작업, 배포 완료)**
- 기존엔 경계를 **장면(시각 컷)**에만 맞춰, 말하는 도중 컷이면 "갑자기 대사 시작"·"말 뚝 끊김"이 났다.
- **발화(문장) 타임스탬프로 스냅**: 시작이 대사 중간이면 문장 처음으로, 끝이 대사 중간이면 문장 끝까지 (2.5초 밖이면 손 안 댐). AI·휴리스틱 두 경로 모두 적용, 단위테스트 6/6 통과.

### 2-3. 학습 파이프라인 + 자동화 ✅

수동 과정(매칭→설명→학습→적용)을 **채널 버튼 하나로**:
```
매칭(오디오) → 구간 설명(Gemini) → 규칙 학습(LEARN) → 채널 프로파일 저장
  → 이후 그 채널 영상 분석에 자동 주입 (신뢰도 낮으면 가중 약하게 = 과적합 방지)
```
- Lab에서 진행 상황·학습된 규칙(신뢰도·표본·규칙 내용)을 화면으로 확인 가능.
- 독립 확증: LEARN이 뽑은 규칙("60초 초과 회피, 25~60초 최적")이 §2-2 채점 결론과 **같은 방향** → 두 다른 경로가 같은 답.

### 2-4. 데이터 자산 (하하PD 채널)

| 항목 | 값 |
|---|---|
| 자동 매칭 쌍 | **72쌍** (오디오 정렬, 사람 확인 전) |
| 성과 tier 분포 | high 26 · mid 24 · low 21 (균형 — 학습에 이상적) |
| 구간 설명 완료 | 40건 |

## 3. 핵심 질문의 현재 답 — "학습이 효과 있나"

### 확실한 것 ✅
- **엔진의 구조적 개선(길이·경계)은 증명됨.** 이건 학습과 무관하게 모든 채널에 유효하다.
- 학습 파이프라인이 **유효한 규칙을 뽑는다**는 것도 확인됨(독립 경로 교차검증).

### 아직 측정 중 ⏳ — 그리고 왜 신중해야 하는가
이번에 **중요한 방법론적 사실**을 발견했다:

> 추천 엔진은 temperature 0인데도 **실행할 때마다 결과가 조금씩 다르다.**
> 같은 조건을 5번 돌리니 Hit@5가 0.00 / 0.33 / 0.33 / 0.67 / 0.67 로 요동쳤다.

즉 **한 번 재서 "학습 켜니 올랐다"고 하면 그게 학습 덕분인지 우연인지 구분할 수 없다.** 그래서:
- **반복 채점기**를 만들어 조건마다 여러 번 반복 → 평균±변동으로 판정하게 했다.
- **홀드아웃을 3편(정답 26건)으로 확대** — 정답이 많아야 우연이 걸러진다.
- **누수 차단**: 학습은 홀드아웃 3편을 제외한 데이터로만 (정답을 미리 본 채로 채점하면 반칙).

**현재 상태:** 홀드아웃 2편(원정대5=정답 18건, 원정대4=5건) 다운로드 완료, 장면 분석 진행 중. 완료되면 3편 반복 A/B로 **"학습 켬 vs 끔"의 Hit@10 평균 차이가 변동(σ)을 넘는가**를 판정한다.

> 📊 **다중 홀드아웃 A/B 결과: 측정 진행 중 — 완료 시 이 자리에 갱신**
> (누수 없는 프로파일 이미 학습됨: 신뢰도 0.75, 훅 가중 차등 — 돌직구 1.56·반전 1.11·웃음 0.82)

---

## 4. 정직한 한계

- Hit율 0→67%는 **홀드아웃 1편·정답 3건**의 단일 측정이다. 엔진 변동을 감안하면 이 숫자 하나로 "확정"이라 쓰면 안 된다(그래서 지금 확대 측정 중).
- 매칭 72쌍은 **아직 사람이 확인하지 않았다**(오디오 정렬 신뢰도는 높지만 표본 검수 필요).
- 채널 **1개**(하하PD)만으로는 "이 방법이 모든 채널에 통한다"는 일반화가 안 된다.
- 원본 롱폼이 없는 채널(이미 잘린 클립만 올리는 편성)에는 오디오 매칭 자체가 성립하지 않는다.
