# STEP-D UI 기획 — 원본→쇼츠 의존성 시각화

> **문제 정의:** 여러 영상을 업로드하고 분석해도 "어떤 원본 영상에서 어떤 쇼츠가 파생되었는지" 의존성이 UI에서 드러나지 않음.  
> **목표:** 원본→분석→추천→쇼츠→배포의 흐름을 한 화면에서 추적 가능하게  
> **작성일:** 2026-07-17

---

## 현재 UI의 문제점 진단

### 현재 구조
```
대시보드 → 콘텐츠(프로그램) → 회차(Episode) → [소스|분석|추천|클립|배포] 탭
```

### 구체적 페인 포인트
| # | 문제 | 원인 |
|---|------|------|
| P1 | 원본 영상을 보려면 '소스' 탭, 쇼츠를 보려면 '클립' 탭 — **맥락 분리** | 탭이 수평적으로 완전히 분리되어 있음 |
| P2 | 쇼츠 카드에 "어느 원본의 몇 분 몇 초 구간인지" 표시 없음 | 클립 목록에 원본 영상 참조 정보 부재 |
| P3 | 여러 회차 업로드 시 **어떤 회차에서 몇 개의 쇼츠가 나왔는지** 한눈에 안 보임 | 회차 목록에 결과물 카운트 없음 |
| P4 | AI 분석 결과(쇼츠 추천)과 실제 채택된 클립 간 연결이 시각적으로 없음 | 분석 탭과 추천/클립 탭이 분리 |
| P5 | 타임라인 기반 탐색 불가 — "이 쇼츠는 원본 영상의 어디쯤인가?"를 알 수 없음 | 영상 플레이어와 쇼츠 목록이 연동 안 됨 |

### 데이터는 이미 있음
`Recommendation`, `Clip` 모두 `episodeId`와 `startTime`/`endTime`을 가지고 있음. `types.ts` 기준:
- `Recommendation.episodeId` + `startTime`/`endTime` → 원본 참조
- `Clip.episodeId` → 원본 참조
- `Episode.pipeline.stage` → 분석 진행 상태
- `AnalysisShort.start`/`end` → AI 쇼츠 추천의 타임스탬프

**UI만 연결해주면 됨.**

## 개선안 1: 회차 상세 — "소스 중심 허브" 레이아웃

### 기존 → 변경

**기존:** 5개 수평 탭 (소스 | 분석 | 추천 | 클립 | 배포)  
**변경:** 좌우 분할 레이아웃 + 하위 서브탭

```
┌──────────────────────────────────────────────────────────────┐
│  [PipelineStrip: 소스→병합→분할→분석→●추천→편집→인코딩→배포]  │ ← 기존 유지
├────────────────────────┬─────────────────────────────────────┤
│                        │  [🔥쇼츠 추천] [🎬파생 클립] [📊분석] │ ← 서브탭
│   원본 영상 플레이어     │─────────────────────────────────────│
│  ┌──────────────────┐  │                                     │
│  │                  │  │  ┌───────────────────────────────┐  │
│  │     VIDEO        │  │  │ 🔥 #1  자막으로 보는 명장면     │  │
│  │                  │  │  │ ⏱ 00:05 ~ 00:35 (30초)       │  │
│  │ ══●══●════●══    │  │  │ 🏷 예능 · appeal 5  │ [채택]  │  │
│  │  타임라인 + 마커   │  │  └───────────────────────────────┘  │
│  └──────────────────┘  │  ┌───────────────────────────────┐  │
│                        │  │ 🔥 #2  유재석 돌발 발언         │  │
│  매체 정보              │  │ ⏱ 12:30 ~ 13:00 (30초)       │  │
│  1920×1080 · 42:15    │  │ 🏷 인터뷰 · appeal 4 │ [채택]  │  │
│                        │  └───────────────────────────────┘  │
│  [다운로드] [재분석]     │  ┌───────────────────────────────┐  │
│                        │  │ 🔥 #3  박명수 표정 변화          │  │
│                        │  │ ⏱ 28:15 ~ 28:55 (40초)       │  │
│                        │  │ 🏷 하이라이트 · appeal 3│[채택] │  │
│                        │  └───────────────────────────────┘  │
└────────────────────────┴─────────────────────────────────────┘
```

### 핵심 인터랙션
1. **쇼츠 카드 호버 → 타임라인 하이라이트:** 카드에 마우스 올리면 영상 타임라인에 해당 구간이 강조 표시됨
2. **쇼츠 카드 클릭 → 영상 점프:** 해당 타임스탬프로 플레이어 이동 + 5초 미리보기 재생
3. **타임라인 마커 클릭 → 카드 스크롤:** 마커 클릭 시 우측 패널이 해당 쇼츠 카드로 자동 스크롤
4. **서브탭 전환:** '쇼츠 추천'(분석 AI 결과 + 추천 탭), '파생 클립'(채택완료 클립), '분석'(장면/자막 원본 데이터)

### 변경 파일
- `episode-detail.tsx` — 전체 레이아웃 변경 (탭 제거, 좌우 분할로)
- 좌측 패널 신규 컴포넌트: `source-panel.tsx` (영상 플레이어 + 타임라인 마커)
- 우측 패널 신규 컴포넌트: `derivatives-panel.tsx` (서브탭 + 카드 리스트)

## 개선안 2: 타임라인 마커 연동

원본 영상 플레이어의 재생 바(progress bar)에 AI 분석 결과를 마커로 오버레이:

```
영상 타임라인 (42:15 기준)
├─────●────────●───────────────●─────────────┤
│     │        │               │             │
│    #1       #2              #3            │
│   30초      30초            40초           │
│  예능/5    인터뷰/4      하이라이트/3      │
```

### 구현 방식
1. `AnalysisShort[]` 또는 `Recommendation[]`의 `start`/`end` 값으로 마커 위치 계산
2. 마커 색상: appeal 점수 기반 (≥4: amber(정예), ≥3: blue, <3: gray)
3. 마커 너비: 구간 길이에 비례
4. 현재 재생 위치가 마커 구간 안에 있으면 해당 마커 + 카드 하이라이트

### 신규/수정 파일
- `timeline-markers.tsx` — 신규 컴포넌트
- `source-panel.tsx` — 영상 플레이어에 마커 통합

## 개선안 6: 타임라인 인라인 트리밍 (In-Place Trim)

### 개념: "타임라인만 잡고 최종 렌더는 마지막"

AI가 추천한 구간을 운영자가 타임라인에서 **직접 드래그로 조정**할 수 있는 기능. 비파괴 편집 — 실제 비디오 커팅 없이 in/out 포인트만 바꾸고, 진짜 렌더링은 "최종 렌더" 버튼을 누를 때 한 번에 실행.

```
예: AI 추천 "3:00~4:30" → 운영자가 4:35까지 늘리고 싶으면
→ 타임라인 핸들을 오른쪽으로 드래그 → in/out 메타데이터만 갱신 → 실시간 미리보기
→ 최종 확정 시 렌더
```

### UI 구조

```
원본 영상 플레이어
┌──────────────────────────────────────────────────┐
│                                                   │
│                    VIDEO                          │
│                                                   │
├────●══════════════●─────────────────────────────┤
│  3:00           4:30                              │
│   ◁─ 드래그 ─▷    ◁─ 드래그 ─▷                   │
│                                                   │
│  [◀ -3초] [현재: 3:00~4:30 (90초)] [+3초 ▶]      │
│  [원본에서 미리보기] [AI 추천으로 리셋] [최종 렌더] │
└──────────────────────────────────────────────────┘
```

### 핵심 인터랙션

| 동작 | 결과 |
|------|------|
| **In 핸들 드래그** | 시작 시간 변경 (앞으로/뒤로). 드래그 중 실시간 프레임 미리보기 |
| **Out 핸들 드래그** | 종료 시간 변경. 최대 원본 끝까지 확장 가능 |
| **-3초 / +3초 버튼** | 프레임 단위 정밀 조정 |
| **"AI 추천으로 리셋"** | 원래 AI 추천 구간으로 복원 |
| **"원본에서 미리보기"** | 현재 설정된 구간 루프 재생 |
| **"최종 렌더"** | 확정된 in/out으로 ffmpeg 트림·인코딩 실행 |

### 시각적 디자인

- **선택 구간**: 반투명 파란색 오버레이 (`rgba(59,130,246,0.3)`)
- **In/Out 핸들**: 흰색 세로 바 + 좌우 화살표 아이콘
- **AI 추천 원본 구간**: 연한 회색 점선 마커 (비교용으로 잔존)
- **드래그 중**: 프레임 썸네일 팝오버 (현재 위치의 장면 미리보기)

### 데이터 모델 (기존 활용)

```typescript
// 기존 Recommendation에 이미 있음 — 이걸 그대로 UI에 바인딩
interface Recommendation {
  startTime: number;  // ← in 포인트
  endTime: number;    // ← out 포인트
  // ...
}

// "최종 렌더" 누르면 adopt API 호출 → 기존 파이프라인 그대로
POST /api/recommendations/:id/adopt
Body: { startTime: 183.5, endTime: 275.0 }  // ← 조정된 값
```

### STEPD만의 차별점

Opus Clip은 AI 추천 그대로 쓰거나 버리는 2택. STEPD는:
1. AI 추천을 **시작점**으로 삼고
2. 운영자가 **타임라인에서 직접 미세 조정**
3. Vision 분석 점수, 자막 구간, 장면 경계를 **참조선**으로 오버레이해서 "이 구간을 늘리면 자막이 끊긴다" 같은 피드백 제공

### 구현 파일

- `trim-handles.tsx` — 드래그 가능한 In/Out 핸들 컴포넌트
- `source-panel.tsx` — 트리밍 모드 통합 (기존 마커 → 편집 모드 전환)
- API: `PATCH /api/recommendations/:id/trim` (메타데이터만 갱신, 렌더 없음)

## 기술 구현 노트

### 데이터 흐름 (이미 존재)
```
Episode ──→ Recommendation[] (episodeId 참조)
Episode ──→ Clip[] (episodeId 참조, startTime/endTime)
Episode ──→ MediaAsset (master 영상)
MediaAsset ──→ MediaAnalysis (AI 쇼츠 추천: AnalysisShort[])
```

`useAppData()`에서 이미 `recsForEpisode()`, `clipsForEpisode()`, `mediaForEpisode()` 제공.

### 타임라인 마커 연동
```tsx
// source-panel.tsx 핵심 로직
const markers = [...analysis.shorts, ...recs].map(item => ({
  start: (item.start ?? item.startTime) / durationSec * 100, // %
  end: (item.end ?? item.endTime) / durationSec * 100,
  appeal: item.appeal ?? 0,
  id: item.id ?? item.rank,
}));
```

### 상태 관리
- 기존 `useAppData()` store 그대로 사용 — 추가 API 호출 불필요
- 서브탭 상태: `useState` 로컬로 충분 (URL 쿼리 연동은 선택)
