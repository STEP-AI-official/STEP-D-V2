# STEP D — Product Vision & Plan

**AI 기반 Media Production & Growth OS**

---

## 한 줄 요약

방송사·제작사·브랜드 콘텐츠팀이 원본 영상을 올리면, 영상 분석 → 편집 제안 → PPL 리포트 → 배포 승인 → 성과 분석 → 다음 콘텐츠 방향까지 처리하는 **AI MediaOps OS**.

핵심은 AI가 최종 영상을 만드는 게 아니라, **영상의 모든 정보를 시간축 기반 데이터로 구조화**하는 것.

---

## Problem

영상 제작 조직의 반복 비용 5가지:

| # | 문제 | 현재 방식 |
|---|------|-----------|
| 1 | 원본 영상 검수 | 처음부터 끝까지 직접 시청 |
| 2 | 편집 포인트 탐색 | 강한 발언·리액션·웃긴 포인트 사람이 직접 찾음 |
| 3 | PPL/브랜드 리포트 | 노출 초수·위치·음성 언급 수동 확인 |
| 4 | 배포 승인 | 메신저·다른 툴로 피드백 주고받음 (타임코드 흩어짐) |
| 5 | 배포 후 성과 분석 | 플랫폼별 지표 수동 정리 |

---

## Solution — 시간축 데이터 구조화

영상 1편이 이런 데이터로 변환됨:

```
00:01:23 — 브랜드 A 로고 노출 후보
00:03:12 — 7초 무음 구간
00:05:44 — 개인정보 노출 의심
00:07:20 — 쇼츠 후보 구간
00:11:02 — 음량 피크 경고
00:15:30 — 경쟁사 로고 후보
```

이 데이터가 검수·편집·PPL 리포트·배포 승인·성과 분석·다음 기획에 **모두 재사용**됨.

---

## Product Architecture (8 Layers)

| Layer | 역할 |
|-------|------|
| **Ingest** | 원본 업로드, 프록시 생성, 메타데이터 추출 |
| **Analysis** | STT, OCR, 장면 분리, 얼굴/객체/로고 탐지, 오디오 분석, 기술 QC |
| **Video Intelligence IR** | 모든 분석 결과를 타임코드 기반 JSON으로 저장 |
| **Review** | 웹 플레이어, 이슈 카드, 타임코드 댓글, 승인/반려/수정 요청 |
| **Edit Suggestion** | 침묵 제거, 문제 장면 컷/블러, 쇼츠·하이라이트 후보 제안 |
| **Distribution Analytics** | 채널별 조회수, 댓글, 참여율, 완주율 수집 |
| **Report** | PPL 리포트, 검수 리포트, 주간 보고서, 다음 제작 방향 제안 |
| **Feedback Loop** | 잘된 영상의 훅·자막·출연자·주제·PPL 패턴 학습 → 다음 전략 반영 |

## MVP Roadmap

### Phase 1 — Review OS MVP ← **현재 구현 중**
- 영상 업로드 (파일 / YouTube 링크)
- AI 자동 분석 (STT, 장면 분리, OCR, 객체/로고 탐지, 기술 QC)
- 타임코드별 이슈 카드 생성
- 웹 플레이어 기반 리뷰
- 담당자 승인/반려/수정 요청

### Phase 2 — PPL Report ← **부분 구현됨**
- 브랜드명/상품명 입력
- 음성 언급·OCR 언급·로고 노출 탐지
- 노출 시간 계산
- 사람이 확인한 PPL 리포트 생성

### Phase 3 — Edit Suggestion
- 무음/중복 구간 제거 제안
- 하이라이트·쇼츠 후보 추천
- 블러/크롭/컷 제안
- 편집안 미리보기

### Phase 4 — Distribution Analytics
- YouTube 중심 성과 수집
- 댓글 수집 및 요약
- 영상별 성과 리포트
- 주간 내부 보고서

### Phase 5 — Feedback Loop
- 잘된 영상의 공통 특징 저장
- 플랫폼별 성공 패턴 분석
- 다음 영상 방향 추천
- 제목/썸네일/자막 스타일 실험 제안

## Target Customers

1. **방송/예능/웹콘텐츠 제작사** — 검수·PPL 리포트·납품 전 QC 비용 절감
2. **브랜드 콘텐츠팀** — 브랜드 세이프티, 캠페인 성과 분석
3. **MCN/크리에이터 스튜디오** — 다수 영상 빠른 편집·배포·댓글 분석
4. **광고대행사/마케팅 에이전시** — 플랫폼별 성과 보고서
5. **교육/강연/행사 영상팀** — 긴 원본에서 핵심 구간 추출 + 검수

초기 타겟 조건: 주당 여러 영상 제작 + 검수/쇼츠 추출에 인력 투입 + PPL 리포트 수동 작성

**STEP D = 이 세 영역을 하나의 OS로 연결**

## Moat

모델 자체가 아니라 **고객의 워크플로우와 데이터**에 방어력이 있음:

1. **Workflow Lock-in** — 검수·승인·리포트·배포 분석이 누적될수록 이탈 어려움
2. **Timecode Data Asset** — 장면·발화·로고·이슈·승인 로그가 시간축으로 쌓임
3. **Creative Performance Dataset** — 어떤 훅/자막/출연자/PPL이 플랫폼에서 잘 먹혔는지 누적
4. **Review & Compliance Log** — 기업 고객용 감사 로그
5. **Customer-Specific Model** — 고객별 브랜드·금칙어·납품 규격 학습 → 정확도 향상
